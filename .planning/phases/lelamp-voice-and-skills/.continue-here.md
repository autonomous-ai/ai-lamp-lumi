<remaining_work>

- **CRITICAL: Voice architecture redesign** — Current flow is broken:
  - OpenClaw is text-based → receives messages via Telegram/chat → responds on ALL channels including Telegram
  - Voice should be LOCAL loop: mic → STT → OpenClaw → TTS → speaker (no Telegram)
  - `voice_service.py` already has Deepgram STT pipeline with "Hey Lumi" wake word
  - It sends transcript to `POST /api/sensing/event {type: "voice", message: transcript}`
  - BUT `/api/sensing/event` endpoint doesn't exist in Go code
  - AND OpenClaw response leaks to Telegram instead of staying local (speaker only)
  - Need to: create sensing/event endpoint, route voice messages as local-only (not multichannel), ensure response goes only to speaker

- Deploy all changes: `make upload-skills` → deploy LeLamp (server.py) → deploy Lumi (Go binary)
- Test end-to-end: voice conversation, scene switching, LED effects, servo aim
- Update docs (`docs/*_vi.md`) per CLAUDE.md rules for all changed APIs
</remaining_work>

<decisions_made>

- Decided TTS is auto (Go layer intercepts responses) + explicit (voice skill for OpenClaw to call directly). Both paths needed.
- `stripForTTS()` cleans markdown/emoji before speaking — OpenClaw responses often contain formatting.
- Skills use "Priority" + "When NOT to use" sections to prevent OpenClaw from choosing wrong skill (root cause of "chói khi ngủ" bug).
- LED effects run in background thread with `threading.Event` stop signal. Starting new effect auto-stops previous.
- AIM_PRESETS angles are placeholder values — need calibration on real hardware.
</decisions_made>

<blockers>

- **Voice architecture**: OpenClaw multichannel design (Telegram, Slack, etc.) conflicts with local-only voice interaction. Need a way to tag voice messages as "local" so responses only go to speaker, not Telegram. This might require changes to OpenClaw's chat.send or a separate "voice session" concept.
- **No `/api/sensing/event` endpoint**: voice_service.py sends transcripts here but the Go endpoint doesn't exist yet. Need to create it and wire it to OpenClaw chat.send.
- **AIM_PRESETS calibration**: Servo angle values are guesses — need testing on real hardware.
</blockers>
